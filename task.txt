Создать RESTful API для посылки смс с функцией аудита отправленных сообщений в реляционной базе данных.

Функции API:
Отправить смс сообщение (с возможностью прикрепления тега(ов))
Отправить группу смс сообщений (с возможностью прикрепления тега(ов))
Получить список ранее отправленных смс сообщений удовлетворяющих критериям поиска (время, телефон, набор тегов)

Язык реализации Java. 
Технологии реализации на выбор соискателя. 
Контракт для  RESTful API на выбор соискателя.
Реляционная база данных и схема данных на выбор соискателя. 
Сервис для физической посылки смс сообщений любой на выбор соискателя (можно использовать http://wiremock.org/ 
для создания фиктивного сервиса).

Вопрос.
 Что значат теги в контексте смс? 
 Они должны быть встроенны в само сообщение или это просто некая запись в бд для упращения фильтрации во время поиска?

Ответ. 
 Тег — идентификатор для категоризации, описания, поиска данных и задания внутренней структуры 
 https://ru.wikipedia.org/wiki/Тег_(метаданные)
 В контексте тестового задания теги могут присваиваться сообщениям во время отправки, 
 а точнее при фиксации факта отправки сообщения в базе данных. 
 Впоследствии с помощью тегов можно находить интересующие сообщения. 
 Например для сообщений о промо компании при отсылке сообщения сообщению может быть присвоен тег "промо", 
 который можно ипользовать для последующих поисков сообщений соответствующих промо компании.
 
Вопрос.
 Объясните, что значит 2 пункт в описании функции приложения "отправить группу смс сообщений"?
 Это отправка одного сообщения для несколько телефонных номеров? 
 Или прям разные сообщения для разных номеров?

Ответ. 
 "отправка одного сообщения для несколько телефонных номеров" это подмножество множества "разных сообщения для разных номеров", 
 принимая во внимание, что API строится для "общего случая", ответ тут очевиден ("разных сообщения для разных номеров")
